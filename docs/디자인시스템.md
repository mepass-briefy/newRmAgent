# 디자인 시스템

> **이 파일의 책임**: 색·타이포·간격·컴포넌트 패턴, UI 원칙
> **이 파일이 다루지 않는 것**: 비즈니스 로직 → `서비스정책.md`, 기능 → `요구사항.md`

## DS-PRINCIPLE-001 디자인 원칙

| 원칙 | 적용 |
|---|---|
| **명료함** | 정보 위계 즉시 인식. 같은 화면에 강조 1~2개만 |
| **중립적 톤** | 과한 장식 금지. B2B 컨설팅 컨텍스트 존중 |
| **데이터 우선** | 표·숫자 가독성 최우선. 폼 입력 마찰 최소화 |
| **모호함 회피** | 색만으로 상태 구분 X (텍스트 라벨 동반) |
| **일관성** | 같은 의미는 항상 같은 색·아이콘·문구 |
| **빈 상태도 정보** | 데이터 없을 때 "왜 비어있는지 + 다음 액션" 안내 |

### 하네스 엔지니어 표식 (POL-PRICING-006)

하네스 멤버임을 시각적으로 알리는 두 가지 요소:

| 요소 | 사양 |
|---|---|
| 카드 좌측 인디케이터 | `border-left: 3px solid #9B87C9` (팀빌딩검토 도트와 동일 보라 계열, 라이트/다크 공통) |
| 이름 옆 배지 | 텍스트 `🔧 하네스`, 배경 `#9B87C9`, 글자 `#FFF`, 굵기 700, padding `2px 6px`, radius 4px, fontSize 9 |

보라(`#9B87C9`)는 기존 `팀빌딩검토` 도트 색을 재사용 — 하네스는 팀빌딩 단계에서 발생하는 개념이므로 의미적으로도 일관됨.

## DS-COMP-001 핵심 컴포넌트

### Btn (버튼)

| variant | 스타일 |
|---|---|
| `primary` | brand 배경 + 흰 글자 + 그림자 |
| `secondary` | brandLight 배경 + brand 글자 + 옅은 테두리 |
| `ghost` | 투명 배경 + textSub 글자 + 회색 테두리 |
| `danger` | 빨강 옅은 배경 + 빨강 글자 |

기본 padding `9px 20px`, fontSize 13, fontWeight 600.

### Inp (인풋)

- 기본 padding `10px 13px`
- 포커스: brand 테두리 + 옅은 그림자 (`brand+"1A"`)
- 라벨: 위 12px 간격, fontWeight 600, color textSub
- 필수 표시: 빨간 별표 `*`

### Card

- 흰 배경 (라이트) / card 토큰 (다크)
- 1px solid cardBorder
- 12px 라운딩
- cardShadow

### Badge (상태 배지)

- 도트 + 텍스트
- 배경: `bg2` (양 모드)
- 도트: 상태별 색상 (DS-COLOR-002)
- padding `3px 10px`, fontSize 11

### Toggle

- 너비 40px, 높이 22px, 라운딩 11px
- ON: brand 배경 / OFF: inputBorder 배경
- 흰색 동그라미 16px, 0.2s transition

### ChipBtn (필터 칩)

- padding `5px 12px`, fontSize 12
- active: brandLight 배경 + brand 글자 + 굵게
- inactive: 투명 배경 + textSub

## DS-COMP-002 레이아웃 패턴

### 페이지 헤더

```
[제목 18px / 700]
[부제 13px / textSub]
... (마진 20~24px 후 콘텐츠)
```

### 탭 헤더

- 1px solid divider 하단
- 활성: brand + 2px solid brand 하단 보더 + 700
- 비활성: textSub + 500

### 사이드바 메뉴 항목

- 활성: brandLight 배경, 좌측 3px brand 인디케이터, 600
- 비활성: 투명, textSub 아이콘

### 통계 카드 (대시보드)

```
┌──────────────────────────┐
│ 라벨 (uppercase, 11px)    │
│ 숫자 (26px, 800)          │  [아이콘 38px 박스]
└──────────────────────────┘
```

## DS-COMP-003 모달 / 드로어 패턴

### 중앙 모달

- 오버레이: `rgba(0,0,0,0.5)`, z-index 1000
- 모달: 흰 배경, 12px 라운딩, `0 20px 60px rgba(0,0,0,0.3)` 그림자
- 최대 너비: 560~1040px (콘텐츠 따라)
- 최대 높이: 90vh, 내부 스크롤
- 헤더: 제목 + 부제 + 닫기 버튼
- 푸터: 우측 정렬 액션 버튼

### 우측 드로어 (예: 시급 계산기)

- 오버레이: 동일
- 드로어: 우측 고정, 너비 580px, 100vh
- 헤더 / 본문(스크롤) / 푸터 영역

### 3단계 다이얼로그 (제안서 모드 선택)

- 헤더에 "STEP X / Y" 표시
- 푸터에 [이전][다음] 또는 [이전][시작]
- 단계별 progressive disclosure

## DS-COMP-004 폼 패턴

### 인풋 그룹

```
[라벨 (12px / textSub / 600) *]
[인풋]
[힌트 (10px / textHint)] (선택)
```

### 그리드

- 2열: `gridTemplateColumns: "1fr 1fr", gap: 12`
- 3열: `gridTemplateColumns: "1fr 1fr 1fr", gap: 10`
- 비대칭: `"1.3fr 1.5fr 0.9fr 1fr 1.1fr auto"` (멤버 행 등)

### 멤버 행 (제안서 편집)

다단 구성:
1. 이름 / 역량 / 삭제
2. 직무 / 등급 / 카테고리 / 계약 (자유 입력 시 직무명 인풋 추가)
3. (시급제) 주h / 최대h / 시급
4. (다개월) 월별 시간 그리드 (브랜드 강조 박스)
