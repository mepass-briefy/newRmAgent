## POL-PRICING-001 두 가지 제공 방식 (Variant Mode)

| 코드 | 명칭 | 정의 |
|---|---|---|
| `subscription` | **구독작업자** | 사람만 투입 (기본값) |
| `ai_h` | **AI+H** | AI Agent + 휴먼 함께 투입 |

두 방식은 공존하며 RM이 고객 상황에 맞게 선택해 제안.

## POL-PRICING-002 가격 산정

### 멤버 시급 계산 규칙

```
멤버 1개월 금액 (hourly)        = weekly_hours × 4 × rate × harnessMul(m)
멤버 1개월 금액 (monthly_fixed) = monthly_amount × harnessMul(m)
멤버 단기 총액 (short_term)     = total_amount × harnessMul(m)   // 첫 달에만 계상

harnessMul(m) = m.is_harness ? 0.5 : 1.0
```

### Variant 합계 계산

| 변종 (variant.mode) | 팀 구성비 합계 | 기간 |
|---|---|---|
| `subscription` | Σ 멤버_월금액 (모두 1.0배) | 표준 기간 |
| `ai_h` | (Σ 일반 멤버 금액) × **0.7** + (Σ 하네스 멤버 금액) | 표준 × **0.7** (30% 단축) |

> **이중 절감 방지**: AI+H의 30% 절감은 *일반 멤버에만* 적용. 하네스 멤버는 이미 시급 50%가 적용되었으므로 0.7 곱셈에서 제외한다. 기간 단축 30%는 모든 AI+H variant에 동일 적용.

### 그 외 항목

| 항목 | 정책 |
|---|---|
| AI Agent 운영비 | 실비 정산, **월 상한 1,000,000원** (별도 라인) |
| RM 시급 | 작업자 수수료에 포함, 별도 청구 없음 |
| RM 직접 투입 시 | 예외적으로 멤버에 추가 가능 (`category="rm"`) |
| VAT | 별도 |

상수: `AI_DISCOUNT_RATE = 0.30`, `AI_DURATION_REDUCTION = 0.30`, `HARNESS_RATE_MULTIPLIER = 0.5`, `token_cap = 1000000`

## POL-PRICING-006 하네스 엔지니어 정책

### 정의

**하네스 엔지니어** = AI Harness(=AI Agent)를 직접 운영하면서 일반 직무를 함께 수행하는 작업자. 한 멤버가 "AI 운영자 + 실무자" 두 역할을 겸한다고 보고, 시급을 일반 시급의 **50%로 산정**한다.

### 가용 조건

- 토글은 **AI+H variant (`variant.mode === "ai_h"`)에서만** 노출.
- `subscription` variant에서는 토글 자체를 숨김 (개념상 AI 없는 팀에 하네스 엔지니어가 존재할 수 없음).

### 시급·금액 규칙

POL-PRICING-002의 `harnessMul(m)` 적용. 핵심:
- 하네스 멤버 시급 = 원래 직무 시급 × **0.5**
- AI+H 30% 절감의 0.7 곱셈에서 **제외** (하네스 멤버는 이미 0.5 적용 → 이중 절감 방지)
- 기간 단축은 일반 AI+H 정책과 동일 (variant 전체에 30% 단축)

### 데이터 모델

```javascript
member.is_harness: bool   // 기본 false
```
별도의 "수행 직무(harness_role)" 필드는 두지 않음 — 멤버의 기존 `role_key` / `category`가 그대로 수행 직무.

### 시각적 구분 (DS-COLOR-002 보라 계열 재사용)

- 멤버 카드 좌측에 3px solid `#9B87C9` 인디케이터
- 멤버 이름 옆 `🔧 하네스` 배지 (브랜드 보라 배경 + 흰 글자)

### AI+H와의 관계

AI+H variant는 두 가지 형태로 운용 가능:
1. **하네스 0명**: 기존 AI+H 동작 (전원 0.7 절감)
2. **하네스 ≥ 1명**: 일반 멤버는 0.7, 하네스는 0.5 (POL-PRICING-002 표 참고)

RM은 고객 상황에 맞게 비율을 조절. 일반적으로 시니어 1~2명을 하네스 엔지니어로 두고 나머지는 일반 멤버로 구성.

## POL-VARIANT-001 안(Plan) 구조

### 제안 구조 종류

| 구조 | 의미 | 사용 시점 |
|---|---|---|
| 단일안 | A안 하나만 | 명확한 단일 방향 |
| A안 + B안 | 업무 범위/개요가 다른 두 가지 | 고객이 두 방향 요구 또는 RM 판단 |

### 안 내부 구성 (variant)

각 안은 1~2개의 variant를 가짐:
- 1개: AI+H 또는 구독작업자 중 하나
- 2개: AI+H + 구독작업자 둘 다

### 고객 선택 단위

- **기본**: 모든 variant 노출 (최대 4개: A1+A2+B1+B2)
- **운영**: RM이 1차 필터링 → 2~4개 자유 선택해 제시
- **선택은 고객이 직접**
