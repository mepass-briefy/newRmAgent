## ID 체계

각 파일의 모든 항목은 고유 ID를 가집니다. 변경/논의 시 ID로 지정하세요.

| 접두어 | 파일 | 예시 |
|---|---|---|
| `REQ-{영역}-{번호}` | 요구사항.md | `REQ-PROPOSAL-001` |
| `POL-{영역}-{번호}` | 서비스정책.md | `POL-PRICING-001` |
| `ACC-{영역}-{번호}` | 계정시스템.md | `ACC-ROLE-001` |
| `DS-{영역}-{번호}` | 디자인시스템.md | `DS-COLOR-001` |

영역 코드:
- 요구사항: `PROPOSAL`, `CUSTOMER`, `BRIEFING`, `WORKER`, `DASHBOARD`, `CALC`, `MONITOR`, `PIPELINE`, `AI`, `DATAMODEL`, `CONTRACT`, `CONSULTING`
- 서비스정책: `BIZ`, `PRICING`, `MESSAGE`, `VARIANT`, `MONITOR`, `DATA`
- 계정: `AUTH`, `ROLE`, `PERM`, `AUDIT`, `DATA`, `EDGE`
- 디자인: `PRINCIPLE`, `COLOR`, `TYPO`, `SPACE`, `COMP`, `ICON`, `MOTION`, `DARK`

## 핵심 용어 (전 문서 공통)

| 용어 | 정의 |
|---|---|
| **AI+H** | AI Agent + 휴먼 팀 구성 |
| **구독작업자** | 휴먼 전용 팀 구성 (기본) |
| **안 (Plan)** | 업무 범위/개요가 다른 독립 제안 단위 (A안/B안) |
| **구성 (Variant)** | 안 내부의 모드별 하위 단위 (1안/2안 = AI+H/구독작업자) |
| **RM** | Resource Manager. GRIDGE의 영업·PM 역할 |
| **GRIDGE** | 회사명. IT 리소스 매칭 컨설팅 회사 |

## 코드 구조

- 단일 파일 React 컴포넌트: `rm_work_board_v2.jsx` (~3,500줄)
- 상태: `useState` + `useContext` (ThemeCtx, RatesCtx, CategoriesCtx)
- 저장: `window.storage` (key-value 영구 저장, 아티팩트 단위 분리)
- 외부 라이브러리: 없음 (순수 React + inline style)
- 현재 버전: **v4.3 · 시간 검토 강화** (하네스 엔지니어 + AI 재제안 + 패턴 학습)

## 변경 시 절차

1. 어느 파일의 어느 ID에 영향이 있는지 식별
2. 해당 ID 섹션 먼저 업데이트
3. 다른 파일에서 해당 ID 참조하는지 검색
4. 코드 변경
